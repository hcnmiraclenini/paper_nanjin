# 回看窗口（Lookback）消融实验 - 完整执行指令

## 📋 当前状态

- ✅ **Lookback=6**：已训练完成，已评估（MAPE约19%）
- ❌ **Lookback=7, 10, 14, 30**：需要训练

**Checkpoint命名规则**：`moe_nanjin_lookback_{lookback}`

## 🎯 实验目标

测试不同的 lookback 值（6, 7, 10, 14, 30），找出最适合的历史数据长度。

## ⚠️ 重要说明

### `summarize_lookback_results.py` 的作用
- **不需要重新训练**
- 只是汇总已有评估结果的脚本
- 从评估结果文件中提取MAPE、MSE、MAE等指标
- 生成对比报告

### 测试不同 Lookback 值需要做什么？
**需要重新训练每个 lookback 值的模型**，因为：
1. 不同的 lookback 值会改变输入序列长度
2. 模型需要学习不同长度的历史模式
3. DistributionShiftExpert 的分布偏移特征窗口 = lookback值（动态变化）

### 步骤3：评估所有模型

```bash
cd /root/data3/huangchanni/moe/moe_nanjin1

# 创建结果目录
mkdir -p ../results/moe_nanjin_lookback_ablation

# 评估每个 lookback 值
for lookback in 6 7 10 14 30; do
    echo "=========================================="
    echo "评估 Lookback = $lookback"
    echo "=========================================="
    
    if [ -f "../checkpoints/moe_nanjin_lookback_${lookback}/best_model_latest.pth" ]; then
        python3 evaluate.py \
            --checkpoint ../checkpoints/moe_nanjin_lookback_${lookback}/best_model_latest.pth \
            --data_dir ../data \
            --batch_size 64 \
            --num_workers 8 > ../results/moe_nanjin_lookback_ablation/lookback_${lookback}_evaluation.txt 2>&1
        echo "✅ Lookback=$lookback 评估完成"
    else
        echo "❌ Lookback=$lookback checkpoint不存在，跳过评估"
    fi
    echo ""
done
```

## ⚠️ 注意事项

1. **GPU使用**：使用GPU 3,4,5，避免与Timer-XL任务冲突
2. **训练样本数**：lookback=30需要更多历史数据，训练样本数会减少
3. **随机种子**：所有实验使用相同随机种子（42）确保可复现性
4. **早停机制**：验证集MAPE在50个epoch内无改善时自动停止
