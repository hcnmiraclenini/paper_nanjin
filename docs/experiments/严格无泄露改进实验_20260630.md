# 严格无泄露改进实验（2026-06-30）

## 1. 实验目的

针对老师提出的“测试集测试时不能泄露训练集和验证集、MAE 指标表现一般、不能直接得出过强结论”的意见，本轮重新检查了数据预处理与选模流程，并完成严格无泄露口径下的改进实验。

本轮目标不是用测试集反复挑模型，而是固定以下原则：

- 训练集：仅用于模型参数学习与 `RobustScaler` 拟合。
- 验证集：用于 checkpoint 选择、集成权重选择和全局尺度校准。
- 测试集：只在最终方案由验证集确定后评估一次。

## 2. 数据与预处理修正

数据文件固定为：

- `data/from_nj.csv`：南京南站到其他站的日粒度客流量。
- `data/to_nj.csv`：其他站到南京南站的日粒度客流量。

两表按 `time` 内连接后得到 846 个共同日期、20 个客流变量。时间顺序划分保持为训练/验证/测试 = 50%/25%/25%，对应 423/211/212 天。

已修正 `src/data_loader.py`：默认仅使用训练集拟合 `RobustScaler`，再分别变换验证集和测试集，避免验证/测试分布进入训练预处理。旧版全量拟合 scaler 仅保留为显式复现实验开关 `fit_scaler_on_full_data=True`，不用于最终主结果。

## 3. 严格口径重训配置

所有候选模型均在修正后的无泄露数据加载器上重新训练，lookback=6，horizon=1，hidden_dim=512，batch_size=64，scene gating 与 regime routing 默认开启。

| 候选 | 分布偏移专家特征 | 训练目标 |
| --- | --- | --- |
| stat-quantile | quantile | 默认多目标 |
| stat-basic | basic | 默认多目标 |
| mae-basic | basic | MAE-aware multi-objective |
| mae-basic-mape | basic | MAE+MAPE 加权 |
| mae-robust | robust | MAE-aware multi-objective |

训练日志：

- `../logs/strict_no_leakage/`

checkpoint：

- `../checkpoints/strict_no_leakage/`

## 4. 验证集选型与集成机制

新增脚本：

- `src/evaluate_validation_ensemble.py`

该脚本执行：

- 读取严格口径重新训练的候选 checkpoint。
- 在验证集上收集每个 checkpoint 的预测。
- 仅使用验证集搜索非负集成权重和全局尺度系数。
- 候选表 `candidate_metrics.csv` 只导出验证集指标，不导出候选测试指标，避免人为根据测试表现筛选模型。
- 权重、尺度固定后，测试集仅评估一次。

最终采用 balanced validation objective，即对 MAPE、MSE、MAE 按旧指标阈值归一化后取平均，避免只追求单一指标。

最终运行配置：

```bash
python src/evaluate_validation_ensemble.py \
  --data_dir data \
  --selection_metric balanced \
  --random_samples 200000 \
  --calibrate_top_k 100 \
  --checkpoint ../checkpoints/strict_no_leakage/mae_robust \
  --checkpoint ../checkpoints/strict_no_leakage/mae_basic \
  --checkpoint ../checkpoints/strict_no_leakage/stat_quantile \
  --checkpoint ../checkpoints/strict_no_leakage/stat_basic \
  --checkpoint ../checkpoints/strict_no_leakage/mae_basic_mape \
  --output_dir ../results/strict_no_leakage_ensemble/all_balanced_final_once
```

## 5. 结果

### 5.1 单模型候选（仅按验证集排序）

| 模型 | 验证MAPE/% | 验证MSE | 验证MAE |
| --- | ---: | ---: | ---: |
| mae-robust epoch19 | 30.39 | 0.045551 | 0.159376 |
| mae-basic epoch7 | 30.87 | 0.049368 | 0.167037 |
| stat-quantile epoch32 | 31.92 | 0.039702 | 0.143088 |
| stat-quantile epoch59 | 32.00 | 0.034768 | 0.135687 |
| stat-quantile epoch61 | 32.02 | 0.033152 | 0.132465 |

说明：该表仅展示验证集表现，用于说明候选模型的互补性。最终论文主结果采用验证集 balanced objective 选出的集成模型，不按测试集挑选单模型。

### 5.2 最终严格无泄露集成结果

| 模型 | 选型口径 | 测试MAPE/% | MSE | MAE |
| --- | --- | ---: | ---: | ---: |
| 旧 MoE-Rail 结果 | 原论文结果 | 19.41 | 0.037700 | 0.144000 |
| Strict RAMR-VE（本文最终） | 验证集选权重+尺度，测试集一次评估 | **18.55** | **0.035640** | **0.138028** |

相对旧结果的改进：

- MAPE：19.41% → 18.55%，降低 0.86 个百分点。
- MSE：0.037700 → 0.035640，降低约 5.46%。
- MAE：0.144000 → 0.138028，降低约 4.15%。

验证集确定的全局尺度系数为 1.018。主要权重来自 `stat_quantile` 与 `mae_robust` 系列 checkpoint，说明分位分布特征和鲁棒分布偏移特征在不同训练阶段提供了互补误差结构。

## 6. 论文写作口径

建议将最终模型命名为：

**Strict RAMR-VE（Validation Ensemble）**

中文表述：

> 为避免测试集泄露，本文进一步采用严格时间顺序实验协议：仅在训练集上拟合归一化器，在验证集上确定 checkpoint、集成权重与尺度校准参数，测试集仅用于最终一次评估。在该协议下，Strict RAMR-VE 在南京南站双向铁路客流数据上取得 18.55% MAPE、0.035640 MSE 和 0.138028 MAE，较原 MoE-Rail 的 19.41%、0.037700 和 0.144000 在三项指标上均取得改善。

需要避免的表述：

- 不能写成“用测试集选择最佳 checkpoint”。
- 不能把验证集集成写成测试集后处理。
- 不能声称模型已证明任意线路或任意时间粒度均泛化；跨数据集结论仍应限定为结构鲁棒性验证。
