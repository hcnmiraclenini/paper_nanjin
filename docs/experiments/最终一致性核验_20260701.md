## 2. 无测试泄露协议

最终主结果采用 Strict RAMR-VE，证据文件为 `../results/strict_no_leakage_ensemble/all_balanced_final_once/ensemble_summary.json`。协议字段明确写明：

> Weights and scale are selected on validation set only; test set is evaluated once after selection.

核验要点：

- 时间顺序划分：训练集 423、验证集 211、测试集 212。
- `RobustScaler` 仅在训练集拟合。
- checkpoint、集成权重和尺度校准参数仅由验证集确定。
- 测试集只在最终方案固定后评估一次。
- `src/eval_all_checkpoints.py` 与 `src/pick_best_test_checkpoint.py` 已默认禁止测试集选模，legacy 测试集扫描必须显式加 `--allow_test_selection`。

## 7. 文本边界

论文与逐条说明应保持以下边界：

- 分布偏移专家只能刻画客流窗口 distribution shift，不能识别极端天气、突发事件或运营管制的具体因果来源。
- Traffic 小时级实验只能说明重构窗口、周期特征和扰动代理后的结构鲁棒性初步验证，不能证明日粒度铁路模型可直接跨分辨率迁移。
- Strict RAMR-VE 的优势限定在当前南京南站双向日粒度客流、当前时间顺序划分和当前对比模型集合下。
- MAE-aware 单模型训练能缓解 MAE 弱项，但最终三指标同步改善来自验证集选择的互补 checkpoint 集成与尺度校准，不是测试集调参。
