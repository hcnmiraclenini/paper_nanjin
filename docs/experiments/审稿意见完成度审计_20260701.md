# 审稿意见完成度审计（2026-07-01）

本审计用于核对两位专家意见是否已经在论文、实验和逐条修改说明中形成闭环。所有结果均基于项目真实数据 `data/from_nj.csv` 与 `data/to_nj.csv`，即南京南站到其他站点、其他站点到南京南站的双向日粒度客流。

## 1. 严格无泄露实验协议

最终主结果采用 Strict RAMR-VE，协议如下：

- 数据按时间顺序划分为训练/验证/测试，不随机打乱。
- `RobustScaler` 仅在训练集拟合，再变换验证集和测试集。
- checkpoint、集成权重和尺度校准参数仅由验证集确定。
- `candidate_metrics.csv` 只导出验证集指标，不导出候选测试指标。
- 测试集只在最终方案固定后评估一次。
- 训练过程快照仅作为 `snapshots_for_diagnostics` 诊断用途，不作为测试集选模入口。

## 4. 仍需保持克制的结论边界

- 可以声明 Strict RAMR-VE 在当前南京南站双向日粒度客流数据上三项指标均优于本轮对比模型。
- 不能声明模型已在所有线路、所有时间粒度、所有交通数据集上稳定优于顶会方法。
- 统计/鲁棒分布特征只能说明客流窗口发生 distribution shift，不能反推出具体极端天气、突发事件或运营管制原因。
- Traffic 实验只能作为重构小时级窗口和周期特征后的结构鲁棒性初步验证；当前仓库尚不能证明已从 Traffic 原始数据重新训练复现。
