# Traffic 跨粒度实验证据边界（2026-07-01）

本文件用于核对专家意见1第5条：“从高铁车站日粒度客流预测到高速公路小时级交通流量预测跨度较大，必须重构时间窗口、周期特征和外部扰动特征，否则不能说明方法适用性。”

## 1. 当前可证明事实

- 论文主结论和最终三指标优势只依赖项目内真实南京南站双向日粒度客流数据：`data/from_nj.csv` 与 `data/to_nj.csv`。
- 主实验采用严格时间顺序划分，归一化器仅在训练集拟合，checkpoint、集成权重和尺度校准参数仅由验证集确定，测试集只用于最终一次评估。
- 当前工作区没有检索到可直接复现实验的 Traffic 原始数据文件，例如 `Traffic.csv`、`traffic.csv`、Traffic HDF5 或 Traffic NPZ。
- 现有 Traffic 表格数值来自既有四预测步长平均结果记录和图表常量，不能等同于本仓库当前可从原始 Traffic 数据重新训练得到的结果。

## 2. 论文表述边界

修订稿必须坚持以下边界：

- 不声称铁路日粒度模型可直接迁移到小时级道路交通数据。
- 不声称 Traffic 结果证明稳定跨域泛化能力。
- 不声称 MoE-Rail 在 Traffic 上所有指标全面优于 PatchTST、TimeMixer、iTransformer 和 FreEformer；当前表中 MSE 最优，但 MAE 不是最优。
- Traffic 只能作为“重构小时级窗口、周期编码和局部扰动代理后的结构鲁棒性补充证据”。
- 若要写成完整可复现实验，必须补齐 Traffic 原始数据、数据哈希、预处理脚本、训练日志、验证集选模记录和最终一次测试结果。

## 3. 后续补跑协议

若后续补齐 Traffic 原始数据，应按以下协议补跑：

| 项目 | 要求 |
| --- | --- |
| 数据 | 保存原始文件路径、来源、形状、缺失值、SHA256 |
| 划分 | 时间顺序 70%/10%/20%，不得随机打乱 |
| 归一化 | 仅训练集拟合 scaler，再变换验证集和测试集 |
| 时间窗口 | 使用小时级 lookback；不得复用铁路日粒度窗口 |
| 周期特征 | 使用小时、星期、周末等小时级可用编码 |
| 扰动代理 | 仅使用回看窗口内波动率、分位数、峰均比等统计量 |
| 选模 | checkpoint、集成权重、尺度校准只能看验证集 |
| 测试 | 最终配置固定后只评估一次测试集 |
| 输出 | 保存逐步长结果、四步长平均表、日志和图 |
