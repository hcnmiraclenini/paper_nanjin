# 门控机制统计重算审计（2026-07-01）

## 重算目的

为回应“短期专家权重偏低”和“门控权重解释不足”的审稿意见，本文重新使用当前论文模型、真实南京南站双向客流数据和严格时间顺序测试集计算专家门控统计。重算结果用于替换论文中旧版硬编码表格，避免专家顺序误读和测试窗口场景样本不足导致的过度结论。

## 数据与协议

- 数据文件：`data/from_nj.csv`、`data/to_nj.csv`
- 数据含义：南京南站到其他站、其他站到南京南站的双向日粒度客流
- 合并后有效共同日期：846 个
- 目标变量：20 个（10 个 from + 10 个 to）
- 时间划分：训练集 423、验证集 211、测试集 212
- 归一化：`RobustScaler` 仅在训练集拟合
- checkpoint：`../checkpoints/paper_experiments/ramr_full_robust/best_model_latest.pth`
- 统计脚本：`src/plot_paper_figures.py`
- 输出文件：`../results/paper_figures_ramr_full/门控统计检验.json`

## 专家顺序核验

当前模型输出的专家顺序为：

| 顺序 | 专家 |
| --- | --- |
| 1 | ShortTerm |
| 2 | LongTerm |
| 3 | DistributionShift |

因此论文表格必须按“短期专家、长期专家、分布偏移专家”解释门控列。旧稿中将前两列解释为“长期、短期”的写法会导致错误结论，已更正。

## 测试集场景样本数

| 场景 | 样本数 |
| --- | ---: |
| 工作日 | 152 |
| 周末 | 60 |
| 节假日 | 0 |

当前严格时间顺序测试窗口不含节假日样本，因此不再报告“节假日 vs 工作日”的显著性检验，也不声称节假日权重抬升在当前测试集上显著。

## 场景门控均值

| 场景 | 短期专家 | 长期专家 | 分布偏移专家 |
| --- | ---: | ---: | ---: |
| 工作日 | 0.5927 | 0.0845 | 0.3228 |
| 周末 | 0.2196 | 0.2011 | 0.5793 |

重算结果显示，短期专家并非在所有场景中权重最低。工作日样本中短期专家平均权重最高，说明模型在常规日粒度预测中主要利用相邻日期连续性与短程校正；周末样本中分布偏移专家平均权重最高，说明非工作日需求结构变化会触发分布偏移建模。

## Welch t 检验

| 对比 | 周末均值 | 工作日均值 | t | p |
| --- | ---: | ---: | ---: | ---: |
| 短期专家权重 | 0.2196 | 0.5927 | -11.4009 | 1.66e-23 |
| 长期专家权重 | 0.2011 | 0.0845 | 3.6465 | 4.99e-04 |
| 分布偏移专家权重 | 0.5793 | 0.3228 | 4.9757 | 2.42e-06 |

分布偏移专家在周末样本上的平均权重显著高于工作日，支持“非工作日场景驱动分布偏移专家激活”的机制解释。短期专家在周末权重显著下降，说明其不是失败模块，而是在常规工作日中承担短程连续性建模，在非工作日分布迁移时让位于分布偏移专家。

## 波动率与短期专家关系

以 `北京南_from_nj` 为代表站点，波动率与短期专家权重的 Pearson 相关为 `r=0.0467`，`p=0.4990`。该结果不支持简单线性叙事“波动越大短期专家权重越高”，因此论文中仅将其作为局部迁移可视化，不作为显著性机制证据。机制证据以场景门控均值、周末/工作日 Welch 检验和专家消融为主。

## Regime 门控对齐补充

新增 `docs/experiments/Regime门控对齐审计_20260701.md` 对预测时可见历史窗口的 regime 指标进行审计。结果显示，日总波动率最高 25% 的样本中短期专家权重均值为 0.6714，高于其余样本的 0.4256，p=2.58e-06；峰均比最高 25% 的样本中分布偏移专家权重均值为 0.5909，高于其余样本的 0.3303，p=1.83e-05。该补充说明：高频相邻日变化和峰值突发强度会触发不同专家，不能写成“所有波动都单调触发同一个专家”。

## 论文修订原则

1. 删除旧版节假日显著性检验结论。
2. 明确当前测试集节假日样本数为 0，不做节假日泛化外推。
3. 纠正专家列顺序，按模型实际顺序解释门控权重。
4. 将“短期专家权重低”的叙事改为“短期专家在工作日高权重、周末让位于分布偏移专家”的场景条件化路由解释。
