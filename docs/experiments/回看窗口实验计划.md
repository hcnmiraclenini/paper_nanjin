# 回看窗口（Lookback）消融实验计划

## 📋 实验目标

测试不同的回看窗口（lookback）值，找出最适合南京南高铁客流预测的历史数据长度。

## 🔍 当前状态

- ✅ 实验脚本已准备：`train_lookback_ablation.sh`
- ✅ 汇总脚本已准备：`summarize_lookback_results.py`
- ✅ **Lookback=6**：已训练完成，已评估（MAPE约19%）
- ❌ **Lookback=7, 10, 14, 30**：尚未训练（checkpoint目录命名规则：`moe_nanjin_lookback_{lookback}`）

## 📊 实验设计

### 测试的 Lookback 值
- **6**：当前基线值
- **7**：一周数据
- **10**：约1.5周数据
- **14**：两周数据
- **30**：一个月数据

### 实验配置（与基线模型完全一致，参考README_ABLATION.txt）
- 模型：基线模型（3个专家：ShortTerm + LongTerm + DistributionShift）
- DistributionShiftExpert：分布偏移特征窗口 = lookback值（动态）
- 训练轮数：500 epochs
- 早停耐心：80 epochs
- 学习率：0.001
- Dropout：0.1
- 其他超参数：与基线模型完全一致

## ⚠️ 重要说明

### `summarize_lookback_results.py` 的作用
- **不需要重新训练**
- 只是汇总已有训练结果的脚本
- 从评估结果文件中提取MAPE、MSE、MAE等指标
- 生成对比报告

### 测试不同 Lookback 值需要做什么？
**需要重新训练每个 lookback 值的模型**，因为：
1. 不同的 lookback 值会改变输入序列长度
2. 模型需要学习不同长度的历史模式
3. DistributionShiftExpert 的分布偏移特征窗口也会随 lookback 变化

# 检查是否有已完成训练的模型

ls -lh ../results/moe_nanjin_lookback_ablation/ 2>/dev/null

### 步骤2：训练缺失的 lookback 值

**当前状态**：
- ✅ Lookback=6：已训练并评估完成（MAPE约19%）
- ❌ Lookback=7, 10, 14, 30：需要训练

**Checkpoint命名规则**：`moe_nanjin_lookback_{lookback}`

#### 选项A：使用实验脚本自动训练（推荐）

```bash
cd /root/data3/huangchanni/moe/moe_nanjin1

# 运行完整实验（会自动跳过已完成的lookback=6，训练7, 10, 14, 30）
nohup bash train_lookback_ablation.sh > lookback_ablation.log 2>&1 &

# 查看日志
tail -f lookback_ablation.log
```

#### 选项B：单独训练每个缺失的值

```bash
cd /root/data3/huangchanni/moe/moe_nanjin1

# 训练 lookback=7
python3 train.py \
    --data_dir ../data \
    --lookback 7 \
    --horizon 1 \
    --batch_size 64 \
    --hidden_dim 512 \
    --dropout 0.15 \
    --num_epochs 300 \
    --patience 50 \
    --learning_rate 0.0008 \
    --weight_decay 0.0001 \
    --lambda_mape 0.5 \
    --lambda_mae 1.0 \
    --lambda_mse 0.5 \
    --lambda_balance 0.01 \
    --lambda_range 0.1 \
    --use_both \
    --num_workers 8 \
    --gpu_ids 3,4,5 \
    --ablation_mode baseline \
    --save_dir ../checkpoints/moe_nanjin_lookback_7

# 训练完成后评估
python3 evaluate.py \
    --checkpoint ../checkpoints/moe_nanjin_lookback_7/best_model_latest.pth \
    --data_dir ../data \
    --batch_size 64 \
    --num_workers 8 > ../results/moe_nanjin_lookback_ablation/lookback_7_evaluation.txt 2>&1

# 同样方式训练 lookback=10, 14, 30
```

### 步骤3：评估所有模型

如果所有模型都已训练完成，评估每个模型：

```bash
cd /root/data3/huangchanni/moe/moe_nanjin1

# 创建结果目录
mkdir -p ../results/moe_nanjin_lookback_ablation

# 评估每个 lookback 值
for lookback in 6 7 10 14 30; do
    echo "评估 lookback=$lookback..."
    python3 evaluate.py \
        --checkpoint ../checkpoints/moe_nanjin_lookback_${lookback}/best_model_latest.pth \
        --data_dir ../data \
        --batch_size 64 \
        --num_workers 8 > ../results/moe_nanjin_lookback_ablation/lookback_${lookback}_evaluation.txt 2>&1
done
```

### 步骤4：汇总结果

```bash
cd /root/data3/huangchanni/moe/moe_nanjin1
python3 summarize_lookback_results.py
```

汇总报告将保存在：`../results/moe_nanjin_lookback_ablation/lookback_ablation_summary.txt`

- 5个 lookback 值总时间：10-20小时
- 评估时间：每个模型约5-10分钟

## 📍 结果保存位置

- **模型checkpoints**：`../checkpoints/moe_nanjin_lookback_{lookback}/`
- **评估结果**：`../results/moe_nanjin_lookback_ablation/lookback_{lookback}_evaluation.txt`
- **汇总报告**：`../results/moe_nanjin_lookback_ablation/lookback_ablation_summary.txt`

## 🎯 实验完成后

1. 查看汇总报告，找出最佳 lookback 值
2. 如果最佳值与当前基线（lookback=6）不同，建议更新基线配置
3. 使用最佳 lookback 值进行后续实验

## ⚠️ 注意事项

1. **GPU使用**：脚本配置使用GPU 3,4,5，避免与正在运行的Timer-XL任务冲突
2. **训练样本数**：lookback=30需要更多历史数据，训练样本数会减少
3. **随机种子**：所有实验使用相同的随机种子（42）确保可复现性
4. **早停机制**：如果验证集MAPE在80个epoch内无改善，将自动停止训练
